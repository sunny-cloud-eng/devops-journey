# Terraform Infrastructure as Code Project

## Overview
Learned Terraform and the Infrastructure as Code workflow by writing declarative configuration and using the full init-plan-apply-destroy cycle.

## What I Did
1. Installed Terraform in Codespaces
2. Wrote a main.tf file using HCL
3. Ran terraform init to initialize the project
4. Ran terraform plan to preview changes
5. Ran terraform apply to create resources
6. Made a change and re-applied (Terraform tracked the difference)
7. Ran terraform destroy to clean up

## Key Concepts Learned
- Infrastructure as Code (IaC): managing infrastructure with code, not manual clicks
- HCL: the readable language Terraform uses
- resource blocks define each piece of infrastructure
- terraform init: sets up the project and downloads providers
- terraform plan: previews changes safely before applying
- terraform apply: creates/updates infrastructure
- terraform destroy: removes everything cleanly
- Terraform tracks state to know what exists

## Why It Matters
IaC makes infrastructure repeatable, version-controlled, reviewable, and auditable - a core DevOps practice.

## Skills Demonstrated
- Writing Terraform HCL configuration
- The full Terraform workflow (init, plan, apply, destroy)
- Infrastructure as Code principles
- Version control for infrastructure
