# AWS CloudWatch Monitoring Project

## Overview
Set up AWS CloudWatch monitoring and created a billing alarm, connecting my existing Site24x7 monitoring experience to AWS-native tools.

## What I Did
1. Explored CloudWatch Metrics for AWS resources
2. Created a CloudWatch Alarm on estimated billing charges
3. Set a threshold to alert when charges exceed a limit
4. Configured an SNS topic for email notifications
5. Confirmed the SNS email subscription
6. Verified the alarm state (OK/In alarm/Insufficient data)

## Key Concepts Learned
- CloudWatch is AWS native monitoring, similar to Site24x7
- Metrics are measurements collected from resources (CPU, requests, connections)
- Alarms watch metrics and trigger actions when thresholds are crossed
- SNS sends notifications (email/SMS) when alarms fire
- Logs centralize application and system logs
- Dashboards provide a visual overview
- CloudWatch alarms can trigger Auto Scaling actions
